Chapter 15 — Networking (Standalone Guide)
Networking

The Networking panel lets groups of connected machines form named virtual networks. Where P2P connections give you raw data pipes between neighbors, a network gives those machines a shared identity, a membership list, coordinated routing, and a unified chat system. You can belong to more than one network simultaneously, and you can belong to a network while also being a plain relay on the chain for machines not in any network.

The Networking section is in the right sidebar, directly above P2P Connection. It starts collapsed. Open it to begin.

Prerequisite
Networking requires at least one active P2P connection. Connect to your left neighbor, your right neighbor, or both using the P2P Connection panel before working with Networking. Everything in the Networking panel depends on the underlying P2P chain being established first.
Overview

A network is a named group of machines layered on top of the physical P2P chain. Machines in a network do not need to be direct neighbors — a machine at position +3 and a machine at position −2 can be in the same network, with the machines between them acting as transparent relays.

Every network has exactly one hub — the machine that created the network and manages membership. All other participating machines are members. The hub maintains the routing table and distributes it to all members so that any member can address any other member by ID.

Networks are session-only. They do not persist after a page refresh. If you refresh, you leave all networks and must reconnect your P2P connections before reforming. Your machine ID (the short identifier shown in the P2P Connection panel) is the only thing that persists across sessions.

Scanning the Chain

Before creating or joining a network, run a scan. The scan propagates a probe in both directions along the P2P chain, collecting machine IDs and handles as it goes, and returns a count of detected machines in each direction.

Click ⬡ SCAN NETWORK at the top of the Networking panel. After a few seconds the topology summary updates:

Topology Summary Examples
1 left · 1 right · 3 total — three machines in a chain, you in the middle

2 left · 2 right · 5 total — five machines, you in the middle of a longer chain

1 left · 1 right · 3 total — same display for a looped ring of three

The scan also discovers any named networks that already exist on the chain. Discovered networks appear in the Available Networks sub-section, where you can request to join them.

A scan runs automatically when a new P2P connection opens. You can re-run it manually at any time with the ⬡ SCAN NETWORK button — useful after the chain topology changes or new machines join.

After a Scan
Once a scan completes, the Create Network sub-section populates with a checkbox list of all detected machines. Each entry shows the machine's handle (or ID if no handle is set) and its signed position relative to you — positive numbers to the right, negative to the left. These checkboxes are your invite list.
Creating a Network

Open the Create Network sub-section. You will see a name field and a list of detected machines.

  1. Type a name in the network name field. Names can be up to 24 characters — letters, numbers, spaces, and hyphens.
  2. Check the boxes next to the machines you want to invite. Each entry shows the machine's handle and position. You can invite any subset of detected machines — you do not have to invite everyone on the chain.
  3. Click ⬡ CREATE NETWORK. Invitations are sent to each selected machine. The status line confirms how many invitations were dispatched.
  4. Wait for responses. Each invited machine will see an invitation panel appear in their Networking section. As they accept or decline, the status updates. Once all responses are received, the network is live and the routing table is distributed to all members.

After creation, the network appears in your My Networks sub-section with a HUB badge. The member list shows handles, positions, and roles. As hub, you are also responsible for approving any join requests that arrive later from machines that were not in the original invite.

You Can Only Create One Network
Each machine can be the hub of at most one named network at a time. You can be a member of additional networks simultaneously, but you can only manage one network as hub. If you want to form a second network, a different machine on the chain needs to be its hub.
Joining a Network

There are two ways to join a network: accept an invitation, or request to join a network you discovered during a scan.

Accepting an Invitation

When another machine invites you, an invitation panel appears near the bottom of the Networking section. The panel shows the network name and the hub's handle. Click ACCEPT to join or DECLINE to refuse. Only one pending invitation is shown at a time.

Requesting to Join

If a scan discovers an existing network you are not in, it appears in the Available Networks sub-section with a REQUEST TO JOIN button. Clicking it sends a join request to the network's hub. The hub sees the request in their Join Requests sub-section and can approve or deny it. You will hear back when the hub responds — either a welcome into the network or a quiet denial.

After Joining
Once you join a network, it appears in My Networks with a MEMBER badge. The member list shows everyone in the network, including their handles and positions. The routing table is distributed by the hub, so messages can reach any member regardless of how many machines they are away from you on the chain.
Hub Responsibilities

The hub is the network's administrative center. It maintains the membership list, distributes the routing table, sends heartbeat signals to keep the network alive, and handles join requests. These functions happen automatically in the background — the hub operator does not need to do anything manually once the network is formed.

Join Requests

When a machine outside the network requests to join, the Join Requests sub-section appears in the hub's Networking panel (it is hidden when empty). Each request shows the machine's handle and its chain position. Click APPROVE to admit the machine or DENY to refuse. Approved machines are added to the membership list and immediately receive the routing table.

Dissolving the Network

To end a network you created, open My Networks, find the network, and click DISSOLVE NETWORK. All members are notified and the network ceases to exist. Members are not disconnected from the P2P chain — only the named network layer is removed.

Hub Departure
If the hub machine refreshes, loses its P2P connections, or otherwise becomes unreachable, members will detect the absence after approximately 15 seconds (three missed heartbeats). An automatic hub election then begins. See Hub Election below.
Members & Roles

The My Networks sub-section shows every network you are involved in, each with its full member list. For each member you can see:

FieldMeaning
HandleThe machine's operator handle, or its raw machine ID if no handle has been set. A ★ marks your own entry.
PositionThe machine's signed chain position relative to you at the time it joined. Positive = right, negative = left. This is informational — routing does not depend on it.
RoleHUB for the network manager. MEMBER for everyone else.

To leave a network you are a member of, find it in My Networks and click LEAVE NETWORK. The hub and remaining members are notified. If you are the hub, the button reads DISSOLVE NETWORK and the entire network ends.

Hub Election

Networks are designed to survive the loss of their hub. If the hub stops sending heartbeats for approximately 15 seconds — roughly three missed intervals — the remaining members start an election to choose a new hub automatically.

The election uses a simple rule: the machine with the lowest machine ID among the remaining members wins. Machine IDs are assigned randomly when a session starts, so there is no meaningful way to predict or engineer the outcome. The winner broadcasts a claim, all other machines acknowledge it, and the new hub begins sending heartbeats and distributing an updated routing table.

From a member's perspective, an election is mostly invisible. There may be a brief period of a few seconds where network messages do not route correctly. Once the new hub is established, normal operation resumes. The new hub's badge updates in every member's My Networks display.

If the Original Hub Returns
If the original hub machine reconnects after an election has completed, it rejoins as a plain member. It does not automatically reclaim the hub role. A new network with that machine as hub requires creating a fresh network.
Chat

The Chat section (below P2P Chat in the sidebar, now renamed from P2P Chat) is a unified messaging panel that shows every machine you can communicate with — direct P2P neighbors and network members — in a single dropdown. Each person appears once regardless of how many ways you are connected to them.

Select a machine from the dropdown to load your conversation history with them. The history persists for the current session; each conversation is separate. Type a message and press SEND or hit Enter.

The chat system routes messages automatically by the best available path. If the recipient is a direct P2P neighbor, the message travels directly. If they are only reachable through a network, it routes through the network's relay chain. You do not need to think about which path is used.

Handles in Chat

Machine names in the chat dropdown and in message history reflect the handle the operator has set for that session. If a neighbor connects before setting their handle, their machine ID appears as a placeholder. Once they type a handle, the dropdown and all existing messages update to show the new name immediately — no refresh needed.

Setting Your Handle
Your handle is set in the text field at the top left of the simulator, below the Loop 2.1 logo. It is shared automatically when you connect to a neighbor. If you update it mid-session, your new handle propagates to all connected machines the next time they refresh their peer list — which happens automatically when network state changes.
Multiple Networks

A single machine can be a member of more than one named network simultaneously, but can only be the hub of one. This allows a machine at a junction in a long chain to bridge two different groups without those groups needing to know about each other.

Each network appears as a separate entry in My Networks. Routing tables are maintained independently per network. A message addressed to a machine in Network A does not interact with Network B's routing, even if some machines are members of both.

In the Chat dropdown, each reachable machine appears once. If a machine is reachable through two networks, the system picks the more direct path automatically. The conversation history with that machine is unified regardless of which network the messages actually traveled through.

Overlapping Membership
When two of your networks share a member, that machine appears once in your Chat dropdown with a label showing the first network that knows about them. This is correct behavior. The routing system handles the deduplication; you just see one entry and one conversation thread.
Refresh & Reset

A page refresh is a full reset. After refreshing:

  • All P2P connections are closed and must be re-established
  • All network memberships are ended — you are no longer in any network
  • All chat history is cleared
  • All SDP handshake blobs in the P2P Connection panel are cleared
  • Your machine ID is preserved (stored in your browser's local storage)
  • Your operator handle is preserved if set before the refresh

The SDP textarea fields are explicitly cleared on load and set to autocomplete="off", so browser form restore does not repopulate stale handshake data. If a P2P connection attempt fails for any reason, a refresh gives you a clean slate with no risk of a previous failed attempt interfering.

Reforming After a Refresh
After refreshing, re-run the P2P handshake with your neighbors (P2P Connection panel), then click ⬡ SCAN NETWORK in Networking to rediscover the chain topology. If the network you were in still has other machines running, request to rejoin via Available Networks or ask the hub to re-invite you.
